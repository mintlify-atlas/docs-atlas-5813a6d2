Petari is a community-driven effort to reverse-engineer Super Mario Galaxy 1 back into readable, compilable C++ source code. The project targets the Korean game versions (`RMGK01` and `RMGK02`) and uses CodeWarrior 3.0a3 to produce output that is binary-identical to the original game executable. No game assets are included — you need an existing copy of the game.

## What is Petari?

Game decompilation is the process of reading a compiled binary and writing equivalent C++ source code that, when compiled with the same compiler and flags, produces a byte-for-byte identical binary. Petari does this for Super Mario Galaxy 1.

The project is **not** a PC port. It is purely a research and preservation effort to understand how the game was originally written.

<Note>
  You must own a legitimate copy of Super Mario Galaxy 1 (Korean version `RMGK01` or `RMGK02`) to build the project. The repository contains no game assets.
</Note>

## Quick start

<Steps>
  <Step title="Install dependencies">
    Install Python, Ninja, and (on non-Windows) Wine. See [Dependencies](/setup/dependencies) for platform-specific instructions.
  </Step>
  <Step title="Clone and extract">
    Clone the repository, then use Dolphin Emulator to extract `sys/main.dol` from your game disc into `orig/RMGK01/`.
  </Step>
  <Step title="Configure and build">
    Run `python configure.py` to generate the build files, then `ninja` to perform the initial analysis and build.
  </Step>
  <Step title="Start decompiling">
    Open `objdiff` and select an unmatched function to begin writing C++ that matches the original assembly.
  </Step>
</Steps>
